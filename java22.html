The Builder design pattern is a creational design pattern that is used to construct a complex object step by step. It separates the construction of an object from its representation, allowing the same construction process to create different representations of the object.

Key Components:

  • Director: The director is responsible for orchestrating the construction process by using a builder.
  • Builder: The builder defines an interface for constructing parts of the object. It may include methods for setting various attributes or properties of the object.
  • Concrete Builder: Concrete builder classes implement the builder interface and provide specific implementations for constructing the object. Each concrete builder can create a different representation of the object.
  • Product: The product is the final object being constructed. It typically has a complex structure and is created using a builder.

Advantages of the Builder Pattern:

  • Separation of Concerns: The pattern separates the construction logic from the representation, making the code more maintainable and flexible.
  • Reusability: Builders can be reused to create different variations of the product without modifying the client code.
  • Fluent Interface: It allows for a fluent and readable construction process, with method chaining to set attributes.
  • Complex Object Creation: Useful when creating objects with numerous configuration options or optional parameters.

Example Implementation:


// Product
class Pizza {
    private String crust;
    private String sauce;
    private String topping;

    public Pizza(String crust, String sauce, String topping) {
        this.crust = crust;
        this.sauce = sauce;
        this.topping = topping;
    }

    // Getters for attributes
    // ...
}

// Builder Interface
interface PizzaBuilder {
    Pizza build();
    PizzaBuilder setCrust(String crust);
    PizzaBuilder setSauce(String sauce);
    PizzaBuilder setTopping(String topping);
}

// Concrete Builder
class MargheritaPizzaBuilder implements PizzaBuilder {
    private String crust;
    private String sauce;
    private String topping;

    @Override
    public Pizza build() {
        return new Pizza(crust, sauce, topping);
    }

    @Override
    public PizzaBuilder setCrust(String crust) {
        this.crust = crust;
        return this;
    }

    @Override
    public PizzaBuilder setSauce(String sauce) {
        this.sauce = sauce;
        return this;
    }

    @Override
    public PizzaBuilder setTopping(String topping) {
        this.topping = topping;
        return this;
    }
}

// Director
class PizzaDirector {
    public Pizza createPizza(PizzaBuilder builder) {
        return builder
            .setCrust("Thin")
            .setSauce("Tomato")
            .setTopping("Cheese")
            .build();
    }
}

public class BuilderPatternExample {
    public static void main(String[] args) {
        PizzaDirector director = new PizzaDirector();
        PizzaBuilder builder = new MargheritaPizzaBuilder();
        Pizza margheritaPizza = director.createPizza(builder);

        System.out.println("Margherita Pizza: " + margheritaPizza.toString());
    }
}

In this example, we have a Pizza class as the product, a PizzaBuilder interface, and a MargheritaPizzaBuilder as a concrete builder. The PizzaDirector orchestrates the construction process, and we create a Margherita pizza using the builder pattern.
