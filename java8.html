In Java, exceptions are categorized into two main types: checked exceptions and unchecked exceptions (also known as runtime exceptions). These two types of exceptions differ in their handling requirements and behavior.

Checked Exceptions:

  • Checked exceptions are exceptions that must be declared or handled explicitly using a try-catch block or by specifying throws in the method signature.
  • These exceptions typically represent external factors that the program should anticipate and handle gracefully.
  • Examples of checked exceptions include IOException, SQLException, and ClassNotFoundException.
  • The compiler enforces the handling of checked exceptions, making it a requirement for the code to compile.

Unchecked Exceptions (Runtime Exceptions):

  • Unchecked exceptions, also known as runtime exceptions, do not need to be declared or explicitly handled in code.
  • These exceptions typically represent programming errors or unexpected conditions within the program.
  • Examples of unchecked exceptions include NullPointerException, ArrayIndexOutOfBoundsException, and ArithmeticException.
  • Unchecked exceptions are not enforced by the compiler, and they can occur at runtime without prior indication.

Differences:

Aspect Checked Exceptions Unchecked Exceptions (Runtime Exceptions)
Handling Requirement Must be declared or handled explicitly. Not required to be declared or handled explicitly.
Typical Use Cases External factors, I/O operations, database interactions, network operations. Programming errors, null references, array index out of bounds, arithmetic errors.
Compiler Enforcement Enforced by the compiler; code won't compile without handling. Not enforced by the compiler; no compile-time requirement.

Example:


// Checked Exception Example
try {
    FileInputStream file = new FileInputStream("file.txt");
    // Code that may throw IOException
} catch (IOException e) {
    // Handle the exception
}

// Unchecked Exception (Runtime Exception) Example
int[] arr = {1, 2, 3};
int result = arr[4]; // This will throw ArrayIndexOutOfBoundsException at runtime

In the checked exception example, we use a try-catch block to handle an IOException, which is a checked exception related to file operations. In the unchecked exception example, we access an array element out of bounds without explicit handling, leading to an ArrayIndexOutOfBoundsException.
