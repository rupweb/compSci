Default methods, introduced in Java 8, are a feature that allows interfaces to have method implementations. Before Java 8, interfaces could only declare method signatures, and implementing classes were required to provide concrete implementations for all interface methods. Default methods provide a way to add new methods to interfaces without breaking existing implementations.

Key Characteristics of Default Methods:

  • Method Implementation: Default methods have method bodies and provide a default implementation for the method.
  • Keyword: Default methods are defined using the default keyword followed by the method signature and implementation.
  • Optional Override: Implementing classes can choose to override default methods if they want to provide their own implementations. If not overridden, the default implementation is used.
  • Interface Evolution: Default methods were introduced to allow interfaces to evolve over time by adding new methods without breaking compatibility with existing classes that implement the interface.

Example of a Default Method:


interface MyInterface {
    void regularMethod(); // Regular abstract method

    // Default method with a default implementation
    default void defaultMethod() {
        System.out.println("This is a default method.");
    }
}

class MyClass implements MyInterface {
    @Override
    public void regularMethod() {
        System.out.println("This is the implementation of the regular method.");
    }
}

public class Main {
    public static void main(String[] args) {
        MyClass myObject = new MyClass();
        myObject.regularMethod(); // Calls the overridden regular method
        myObject.defaultMethod(); // Calls the default method
    }
}

In this example, the MyInterface interface defines a default method defaultMethod() with a default implementation. The MyClass class implements the interface and overrides the regular method regularMethod(). When we create an instance of MyClass and call methods, it demonstrates the use of the default method.

Use Cases for Default Methods:

Default methods are commonly used in Java libraries and APIs to add new methods to interfaces without breaking existing code. They are particularly useful when extending existing interfaces to accommodate new functionality while maintaining backward compatibility.
