In Java, the synchronized keyword is used to create synchronized blocks or methods to ensure that only one thread can access a particular block of code or method at a time. It is a fundamental tool for managing thread safety and preventing race conditions in multithreaded programs.

Key Concepts:

  • Thread Safety: Ensuring that shared resources or critical sections of code can be accessed safely by multiple threads is essential to prevent data corruption and unpredictable behavior.
  • Critical Sections: These are code sections or methods where data is shared among multiple threads. Critical sections need to be protected to avoid conflicts.

Usage of the synchronized Keyword:

The synchronized keyword can be applied in two ways:

  1. Synchronized Blocks: You can use synchronized blocks to wrap a specific section of code to ensure that only one thread can execute that block at a time.
    2. 
synchronized (lockObject) {
    // Synchronized code block
    // Only one thread can execute this block at a time
    // Access shared resources safely here
}
  2. Synchronized Methods: You can declare entire methods as synchronized, which means that only one thread can execute the method on the same instance at a time. This is useful for ensuring that multiple threads don't interfere with each other when accessing object methods.
    3. 
public synchronized void synchronizedMethod() {
    // Synchronized method
    // Only one thread can execute this method at a time
    // Access shared resources safely here
}

Benefits of Using synchronized:

  • Thread Safety: synchronized blocks and methods prevent race conditions and ensure that shared resources are accessed safely by multiple threads.
  • Ordered Execution: synchronized code blocks or methods are executed in a well-defined order, ensuring predictable behavior in multithreaded programs.
  • Preventing Deadlocks: Proper use of synchronized can help avoid deadlock situations where threads block each other indefinitely.

Example:


public class SynchronizedExample {
    private int sharedCounter = 0;

    // Synchronized method
    public synchronized void incrementCounter() {
        sharedCounter++;
    }

    public static void main(String[] args) {
        SynchronizedExample example = new SynchronizedExample();

        // Create multiple threads that increment the shared counter
        Thread thread1 = new Thread(() -> {
            for (int i = 0; i < 1000; i++) {
                example.incrementCounter();
            }
        });

        Thread thread2 = new Thread(() -> {
            for (int i = 0; i < 1000; i++) {
                example.incrementCounter();
            }
        });

        // Start the threads
        thread1.start();
        thread2.start();

        try {
            // Wait for both threads to finish
            thread1.join();
            thread2.join();
        } catch (InterruptedException e) {
            e.printStackTrace();
        }

        // Print the final value of the shared counter
        System.out.println("Shared Counter: " + example.sharedCounter);
    }
}

In this example, a class SynchronizedExample contains a shared counter and a synchronized method incrementCounter(). Two threads, thread1 and thread2, increment the shared counter concurrently. The synchronized keyword ensures that only one thread can execute the incrementCounter() method at a time, preventing data corruption. After starting both threads and waiting for them to finish using join(), the final value of the shared counter is printed. The use of synchronized ensures that the counter is incremented safely, and the final result is consistent.
