The Singleton design pattern is a creational design pattern that ensures a class has only one instance and provides a global point of access to that instance. It is often used when you want to control the instantiation of a class and limit it to a single instance throughout the application's lifecycle.

Key Characteristics of the Singleton Pattern:

  • Private Constructor: The Singleton class has a private constructor to prevent external instantiation.
  • Static Instance: It maintains a static instance of itself, which is the sole instance of the class.
  • Lazy or Eager Initialization: The instance can be lazily or eagerly initialized, depending on the specific requirements.
  • Global Access: The Singleton instance is accessible globally through a static method, allowing other parts of the application to access it easily.

Example Implementation:


public class Singleton {
    // Private static instance variable
    private static Singleton instance;

    // Private constructor to prevent external instantiation
    private Singleton() {
        // Initialization code here
    }

    // Public static method to access the Singleton instance
    public static Singleton getInstance() {
        // Lazily initialize the instance if it doesn't exist
        if (instance == null) {
            instance = new Singleton();
        }
        return instance;
    }
}

public class SingletonExample {
    public static void main(String[] args) {
        // Get the Singleton instance
        Singleton singleton1 = Singleton.getInstance();
        Singleton singleton2 = Singleton.getInstance();

        // Check if both instances are the same
        if (singleton1 == singleton2) {
            System.out.println("Both instances are the same. Singleton pattern works!");
        } else {
            System.out.println("Singleton pattern failed!");
        }
    }
}

In this example, the Singleton class follows the Singleton design pattern. It has a private constructor and a static getInstance() method that returns the Singleton instance. The instance is lazily initialized, meaning it is created only when requested for the first time.

The SingletonExample class demonstrates how to use the Singleton pattern by getting two instances of the Singleton and checking if they are the same instance. If they are the same, it confirms that the Singleton pattern is working correctly.
