Polymorphism is a fundamental concept in object-oriented programming. It describes a paradigm in which a method can have multiple implementations depending on the object that invokes it. In simple terms, polymorphism allows objects of different classes to be treated as objects of a common superclass.

Types of Polymorphism:

  • Compile-time Polymorphism: Achieved through method overloading.
  • Runtime Polymorphism: Achieved through method overriding, where the method to be invoked is determined at runtime.

Example of Polymorphism in Java:

Consider an example with a superclass Animal and its subclasses Dog and Cat, each with their own implementation of the makeSound method:


public class Animal {
    public void makeSound() {
        System.out.println("Some sound");
    }
}

public class Dog extends Animal {
    public void makeSound() {
        System.out.println("Bark");
    }
}

public class Cat extends Animal {
    public void makeSound() {
        System.out.println("Meow");
    }
}

public class TestPolymorphism {
    public static void main(String[] args) {
        Animal myDog = new Dog();
        Animal myCat = new Cat();
        myDog.makeSound();  // Outputs "Bark"
        myCat.makeSound();  // Outputs "Meow"
    }
}

In this example, both Dog and Cat are subclasses of Animal. Each subclass provides its own implementation of the makeSound method. Through polymorphism, an Animal reference can be used to invoke makeSound, and the correct version of the method (depending on whether the object is a Dog or a Cat) is called at runtime.
