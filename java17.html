In Java, both FileReader and BufferedReader are used for reading text files, but they serve different purposes and offer different features.

FileReader

  • Purpose: FileReader is used to read characters from a file.
  • Character-Level Reading: It reads one character at a time, which can be inefficient for large files.
  • Performance: It does not provide buffering, so it may result in slow I/O operations as it reads characters directly from the file.
  • Usage: Typically, FileReader is used when simple character-level reading is required, and performance is not a critical concern.

BufferedReader

  • Purpose: BufferedReader is used to read text from a character input stream with efficient buffering.
  • Buffering: It reads text in chunks (lines or specified buffer size), making it more efficient for I/O operations, especially when dealing with large files.
  • Performance: It offers improved performance compared to FileReader because of buffering.
  • Usage: BufferedReader is commonly used when reading text files efficiently. It is especially useful for reading lines of text.

Example:


import java.io.BufferedReader;
import java.io.FileReader;
import java.io.IOException;

public class FileReaderVsBufferedReader {
    public static void main(String[] args) {
        try {
            // Using FileReader
            FileReader fileReader = new FileReader("sample.txt");
            int character;
            while ((character = fileReader.read()) != -1) {
                System.out.print((char) character);
            }
            fileReader.close();

            // Using BufferedReader
            BufferedReader bufferedReader = new BufferedReader(new FileReader("sample.txt"));
            String line;
            while ((line = bufferedReader.readLine()) != null) {
                System.out.println(line);
            }
            bufferedReader.close();
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

In this example, both FileReader and BufferedReader are used to read text from a file named "sample.txt." You can see the difference in usage and performance between the two.
