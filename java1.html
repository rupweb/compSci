== Operator

Purpose: Used for reference comparison for objects, and value comparison for primitive types.
Usage with Primitives: Compares actual values of primitive data types.
Usage with Objects: Checks if both references point to the same object in memory, not comparing contents.
Example: 

String a = new String("Hello"); String b = new String("Hello"); System.out.println(a == b); // False

.equals() Method

Purpose: Used for content comparison of objects.
Usage: Compares data or content of the objects, depending on implementation.
Default Behavior: In the Object class, it uses == for reference comparison, but often overridden.
Example: 

String a = new String("Hello"); String b = new String("Hello"); System.out.println(a.equals(b)); // True

Key Differences

  • Comparison Basis: == compares references or primitive values. .equals() compares object contents.
  • Overriding: .equals() can be overridden; == cannot.
  • Null Safe: == is null safe, while .equals() may throw NullPointerException if used on a null reference.
